Project Overview & Details

As a long-time webcomic artist but someone slightly newer to web design, I’ve already made several websites for my comics. However, for this one, I really wanted to take the time to make sure it was distinctive and appropriately themed to the story. This website went through a lot of iteration before going live, as well as a few small updates after that as I learned more and become more confident in my design choices.

Under the Rosebed is a slice-of-life comic about three ghosts trying to raise an abandoned child; made up of short, stand-alone stories. Pages are a consistent size, and most of the stories are set in the catacombs, or at night - so I knew I wanted a darker overall theme and colours that would compliment the pages.

The comic is hosted on comicfury, a free platform that offers full customization while handling things like organizing the comics, chapters, and comments itself, making it a very convenient platform.

Project Details

Tools
  • Comicfury Code Editor
  • Clip Studio Paint (for quick mockups, and to create site assets)

Timeframe: ~2 months

Starting Point: Comicfury and the template

Comicfury allows full HTML and CSS customization, along with importing any assets and additional code you’d like.

For this website, I chose to start with one of their templates, to be sure that my website included all the necessary pages and functions. This layout is called “Once I was a King”, and I started with this one mainly because I like the menu style and comic display.

It is common practice to start with a site’s mobile design; however, the larger, full-size view of comic pages is more important here, so I chose to start with the desktop layout and move from there.

The actual colour palettes are very similar, with the dark theme simply having slightly less saturated colours and lower contrast. However; in practice, the inverted text containers make a big difference in perceived brightness, and make reading on a dark screen easier.

Typography

  • Title (Protest Revolution)

  • Headers (Comfortaa Medium)

  • Menu Items (Grenze Gotisch)

  • Body - lorem ipsum (Saria)

I did a lot of wavering on fonts, especially for the menu items, but I feel the ones I chose capture the correct atmosphere and feel while also being very easy to read.

Grenze Gotisch is very decorative, but the gothic, castle-y feel is appropriate, and as the menu items are large and don’t change, it isn’t too difficult to read.

Saira is an excellent body font, with great readability and a huge variety of weights to choose from. The slightly squared shape all the letters have adds a noticeable character to it, and sets it apart from many other body font options.

Layout Changes & Editing the Template

The Overview Page

For all my comic sites, I like to have a most recent page preview on the landing overview page, rather than having the entire latest page, as is comicfury’s default.

Since this comic has short stories, I also wanted an option for users to go directly to the current story’s chapter in the archive. The two preview buttons are in a flexbox together, and stack vertically once the page is narrow enough. On hover, the gradient fades and they expand to double height, allowing the user a clearer preview.

The Archive

The default archive style is a list of comic titles, dates, and comment counts. Because all comic pages are a uniform size, I really wanted to instead have chapters be a set of thumbnails.

In order to make this work properly on all screen sizes, without any big gaps, I went through several HTML/CSS solutions before settling on one I liked best: responsive breakpoints that adjust the percentage width of all divs within the parent container. This way, the comic thumbnails always nicely fill the page while also being allowed to wrap, and I don’t have to do any adjusting as new pages are added.

Comic Display

The default comic display of this template was built very nicely, but I wanted to have something a bit more stylistic. My first change was to include arrows both above and below the comic image (which makes skipping through pages much easier).

Next, I incorporated the navigation into the comic image display itself. Wrapping the nav around the comic made the image display feel more cohesive, and rounding the corners on the nav but not the image itself helps keep with the page’s overall rounded styling while not rounding the corners on the art itself.

A functional change I made was navigation oriented as well - by default, clicking the comic image takes you to the next page. I set up a pair ot buttons so that instead, clicking the left side of the page takes you to the previous one, and the right side takes you to the next.

Additionally, the central rose icon navigates back to the archive for convenience, although it is primarily decorative.

Comment Sections

Comicfury’s native comment system is excellent; very easy to use and manage. However, it does come with some restrictions, and a few layout decisions I’ve never liked.

The most prominent and immediate change I always make to my comment sections is moving the comment box. By default, it’s placed above all comments - this makes sense, as it should be easy to find. However; I’ve always felt it makes the most sense to have after the comments.

The way UtR’s website is designed, the box is beneath the comments, at the bottom. If there are more than three user comments, a button appears right beneath the author note that users can click to jump directly to the box. This means that it’s always easy to access the comment box, but it also doesn’t interfere with reading user comments.

Another change I had to make - comicfury’s comment box is actually an iframe, and can only be styled manually from your manage comic page. This means you get one set of colours for it.

But, this was an issue for me, as the reply box is nested into my comments, and the box would not match their colour. I resolved this using CSS filter, to keep a cohesive look. The default text box is on the left, and the filtered one on the right.

Mobile Compatibility

Most comicfury templates don’t come with mobile compatibility, instead relying on comicfury’s default scroll view layout.

Responsive design is important to me, so I built the entire design to ensure it looks great on any screen, and made modifications to the template and CSS accordingly.

Header & Menu

The header and menu has a couple of different versions, based on screen size.

On XL screens, the header area has a set width, as it spanning the full screen looked out of place when the comic pages and other content did not. On most laptops, it should span the entire screen.

On smaller tablets, the small character icon is hidden to conserve space.

And on phones, the title is compressed, and the menu is collapsed into a hamburger.

Archive

The thumbnail archive is something that was very important to me, and as mentioned, is responsive - however, on very small screens such a s phones, it quickly becomes cumbersome. On tablets, I chose to keep the thumbnails, but on phones, the archive switches to a more easy to navigate and efficient text based list.

Cast Page

On desktop, the cast pages have a large image to the left, and content to the right. On smaller screens, they’re changed to stack vertically, with a banner style image the responsively shrinks horizontally to a square on mobile, and then scales down uniformly based on screen size.

Final Design Changes

By this point, the website was mostly completed. I actually had it live like this for a while, but decided that the all black header had a sort of overpowering, heavy feeling. I wanted to incorporate the titular roses somehow, and add a bit more diversity to the site themes, so I decided to try out a graphic header.

I feel the day and night skies really enhance the light and dark mode, allowing users to have a little more colour while still keeping the overall dark palette of the site.

Conclusion

This project went smoothly, resulting in a functional and unique website, perfectly tailored to the content hosted on it. I tested out some very basic javascript to enhance functionality, and spent more time creating smooth CSS transitions for things like buttons and menus to really polish the site as a whole.

There's nothing about the finished product that I'd do differently; however, I'd probably spend a bit more time re-organizing the template's base code to make the final code cleaner and more consistent. I will likely continue to make small quality of life and feature updgrades to the site as time goes on, to make the user experience even more comfortable.
