Project Overview & Details

Cat Tracks is an app inspired by the walks I used to go on with my old roommate, where we’d look for local cats in the area. Over time, we got to know where a lot of them lived. I thought an app based around taking cute cat photos and walking would be a fun way to share this experience with others, while encouraging users to get more activity.

The brief was fairly open-ended, so I chose something I thought would be engaging and unique to work on for this project. My goal was to create a friendly, easy-to-use app that allows users to learn about, and locate their local cats. Additionally, I had to keep user security in mind while designing the app.

Project Details

Tools
  • Figma

Timeframe: ~3 months

Initial Approach

I started with rough wire frames, and moved through to higher fidelity, getting user feedback along the way to ensure users were able to navigate and interact comfortably with the app. I did a very loose home screen in figma to start, as I find this helps me visualize what I want best, and then moved into wireframes for the rest of the app.

After wireframes and user flows, I moved into my first pass at low-mid fidelity frames in Figma, which is where I work out most of the kinks in layout and figure out things like scale and proportion, as well as fonts. I like to do this early on to make sure the app has the feeling I’m looking for.

Typography

  • Title (Jua)

  • Headers (Fredoka - Semibold)

  • Body (Fredoka)

I kept the typograhpy for the app rather simple, using varying weights, sizes, and colours of Fredoka for all text. It's a friendly, highly ledgible font that adds a lot of character, and has a wide variety of weights.

The app's title is in Jua, which is a similar font with a bit more energy and playfullness. It has too much character to be a body font, but plays well with Fredoka.

Challenges & Solutions

The most challenging part of this app was definitely prototyping. There were several functions I was unsure how to recreate, the large number of screens became overwhelming rather quickly.

This was resolved with time and experimentation, and I’m confident my next large and complicated prototype will go smoother.

Additionally, I had some issues deciding how to re-organize the menu options after user testing. There were too few to have a satisfying hamburger menu without hiding needed info, but too many to easily place.

I believe the final option I decided on considered user feedback while keeping the main functions of the app functional - user menu items within user preferences under the user icon, and the map icon next to the walk button to help prevent confusion between the two.

Final Showcase

The final product came together quite well as a whole! With more time, I’d like to polish things further - but as a MVP and first version, I think it’s captured the desired friendly look well while keeping the main functions of viewing cats and logging walks clear and simple.

For a better look at all screens, check out the working figma prototypes at the end of the case study!

Conclusion

This project went smoothly, resulting in a functional and unique website, perfectly tailored to the content hosted on it. I tested out some very basic javascript to enhance functionality, and spent more time creating smooth CSS transitions for things like buttons and menus to really polish the site as a whole.

There's nothing about the finished product that I'd do differently; however, I'd probably spend a bit more time re-organizing the template's base code to make the final code cleaner and more consistent. I will likely continue to make small quality of life and feature updgrades to the site as time goes on, to make the user experience even more comfortable.
